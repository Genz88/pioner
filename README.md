Проект: Управление пользователями и счетами

Этот проект RESTful API для управления пользователями, их email, телефонами и счетами. Также включена аутентификация через JWT и документация Swagger.

На момент текущей версии проект находится в стадии активной разработки. Работа над ним проводилась в течение неполных двух дней, и в силу ограниченного времени некоторые аспекты требуют доработки. В частности:

- Требуется расширение и улучшение системы валидации.
- Возможны улучшения в части обработки ошибок.

=== Требования ===
Для запуска проекта вам понадобится:
- Java 17 или выше
- Maven (для сборки проекта)
- Git (для клонирования репозитория)
- IDE (например, IntelliJ IDEA) или терминал

=== Клонирование проекта ===
1. Клонируйте репозиторий с помощью команды:
   git clone https://github.com/your-username/your-repository-name.git 
2. Перейдите в директорию проекта:
   cd <your-repository-name>

=== Настройка окружения ===
1. Убедитесь, что у вас установлены все необходимые зависимости:
   - Java Development Kit (JDK) версии 17 или выше.
   - Maven для сборки проекта.

2. Настройте базу данных:
   - PostgreSQL, измените файл application.properties в папке src/main/resources.

   Пример конфигурации:
    spring.datasource.url=jdbc:postgresql://localhost:5432/test8_database
    spring.datasource.username=testuser
    spring.datasource.password=testuser
    spring.jpa.hibernate.ddl-auto=update
    spring.jpa.show-sql=true 

=== Сборка проекта ===
1. Соберите проект с помощью Maven:
   mvn clean install

2. Убедитесь, что сборка завершилась успешно. Если возникли ошибки, проверьте логи и устраните проблемы.

=== Запуск приложения ===
1. Запустите приложение с помощью команды:
   mvn spring-boot:run

2. Приложение будет доступно по адресу:
   http://localhost:8080

3. Документация API (Swagger UI):
   http://localhost:8080/swagger-ui.html

=== Использование API ===
Аутентификация:
1. Для входа используйте эндпоинт:
   POST /auth/login
   Пример тела запроса:
   {
     "username": "user",
     "password": "password"
   }

2. Полученный JWT-токен необходимо добавить в заголовок Authorization для последующих запросов:
   Authorization: Bearer <your-jwt-token>

Примеры запросов:
1. Перевод денег между счетами:
   POST /accounts/transfer
   Параметры:
   - toUserId: ID получателя
   - amount: Сумма перевода

2. Добавление email:
   POST /users/{userId}/emails
   Параметры:
   - email: Новый email

3. Удаление email:
   DELETE /users/{userId}/removeEmail
   Параметры:
   - email: Email для удаления

=== Тестирование ===
1. Запустите тесты с помощью Maven:
   mvn test

2. Убедитесь, что все тесты проходят успешно.
